# groundwork CLI — Usage Guide

`groundwork` analyses a project, detects AWS SDK usage, and generates a
[Terragrunt](https://terragrunt.gruntwork.io/) scaffold with the corresponding
Terraform resources — in a single command.

---

## Installation

**From source:**
```bash
go install github.com/groundwork-dev/groundwork/cmd/groundwork@latest
```

**From GitHub Releases:**
Download the binary for your platform from the
[Releases page](https://github.com/groundwork-dev/groundwork/releases).

**Build locally:**
```bash
make build       # → ./bin/groundwork
make build-all   # → ./bin/groundwork + ./bin/groundwork-mcp
```

---

## Flags

```
groundwork [flags]

  --input              Path to the project to analyse          (default: ".")
  --output             Directory for generated output          (default: "./output")
  --provider           Cloud provider plugin                   (default: "aws")
  --state-bucket       S3 bucket name for Terraform state      (required)
  --state-region       AWS region of the state bucket          (default: "us-east-1")
  --state-lock-table   DynamoDB table name for state locking   (default: "<bucket>-lock")
  --state-encrypt      Enable SSE-S3 encryption on state       (default: true)
  --dry-run            Print what would be generated without writing any files
```

## Examples

### Analyse a Python project

```bash
groundwork \
  --input  ./examples/python_api \
  --output ./examples/python_api/infra \
  --state-bucket my-tf-state
```

Output:
```
groundwork: scanned 18 file(s) from "./examples/python_api"
groundwork: 1 resource spec(s) collected
groundwork: output written to "./examples/python_api/infra"
```

### Dry-run — preview without writing

```bash
groundwork \
  --input  ./my-project \
  --output ./infra \
  --state-bucket my-tf-state \
  --dry-run
```

Output:
```
groundwork: scanned 12 file(s) from "./my-project"
groundwork: 3 resource spec(s) collected
groundwork: dry-run — no files written
  ./infra/terragrunt.hcl                    (root remote_state)
  ./infra/modules/s3/terragrunt.hcl         (aws_s3_bucket)
  ./infra/modules/dynamodb/terragrunt.hcl   (aws_dynamodb_table)
  ./infra/modules/sqs/terragrunt.hcl        (aws_sqs_queue)
```

Use dry-run to verify detection before committing the output directory.

### Custom region and lock table

```bash
groundwork \
  --input  ./my-project \
  --output ./infra \
  --state-bucket    prod-tf-state \
  --state-region    eu-west-1 \
  --state-lock-table prod-tf-state-lock \
  --state-encrypt   true
```

## Generated output

For a project using S3, DynamoDB and SQS, groundwork generates:

```
infra/
├── terragrunt.hcl              ← root: S3 remote state configuration
└── modules/
    ├── s3/
    │   ├── terragrunt.hcl      ← Terragrunt module wrapper
    │   └── main.tf             ← Terraform resource definition
    ├── dynamodb/
    │   ├── terragrunt.hcl
    │   └── main.tf
    └── sqs/
        ├── terragrunt.hcl
        └── main.tf
```

### `infra/terragrunt.hcl` (root)

```hcl
# Generated by groundwork — do not edit manually.

remote_state {
  backend = "s3"

  config = {
    bucket         = "my-tf-state"
    key            = "${path_relative_to_include()}/terraform.tfstate"
    region         = "us-east-1"
    encrypt        = true
    dynamodb_table = "my-tf-state-lock"
  }

  generate = {
    path      = "backend.tf"
    if_exists = "overwrite_terragrunt"
  }
}
```

### `infra/modules/s3/terragrunt.hcl`

```hcl
# Generated by groundwork — do not edit manually.
# Resource type: aws_s3_bucket

include "root" {
  path = find_in_parent_folders()
}

inputs = {
  # TODO: fill in required inputs for aws_s3_bucket
}
```

### `infra/modules/s3/main.tf`

```hcl
# Generated by groundwork — do not edit manually.

terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

resource "aws_s3_bucket" "main" {
  bucket = var.bucket_name
  tags   = var.tags
}

resource "aws_s3_bucket_versioning" "main" {
  bucket = aws_s3_bucket.main.id
  versioning_configuration {
    status = "Enabled"
  }
}

resource "aws_s3_bucket_server_side_encryption_configuration" "main" {
  bucket = aws_s3_bucket.main.id
  rule {
    apply_server_side_encryption_by_default {
      sse_algorithm = "AES256"
    }
  }
}

variable "bucket_name" {
  description = "Name of the S3 bucket"
  type        = string
}

variable "tags" {
  description = "Tags to apply to resources"
  type        = map(string)
  default     = {}
}

output "bucket_id" {
  description = "The name of the bucket"
  value       = aws_s3_bucket.main.id
}

output "bucket_arn" {
  description = "The ARN of the bucket"
  value       = aws_s3_bucket.main.arn
}
```

## Supported languages

| Language | Detection strategy |
|----------|--------------------|
| Go | `github.com/aws/aws-sdk-go-v2/service/<service>` imports |
| Python | `boto3.client('<service>')` / `boto3.resource('<service>')` calls |
| Node.js | `@aws-sdk/client-<service>` in `package.json` or `require`/`import` |

---

## Supported AWS services

| Service | Terraform resource | Generated files |
|---------|--------------------|-----------------|
| `s3` | `aws_s3_bucket` | bucket + versioning + SSE |
| `dynamodb` | `aws_dynamodb_table` | PAY_PER_REQUEST table |
| `sqs` | `aws_sqs_queue` | queue with retention config |
| `sns` | `aws_sns_topic` | topic |
| `lambda` | `aws_lambda_function` | function with env vars |
| `rds` | `aws_db_instance` | RDS instance with variables |
| `secretsmanager` | `aws_secretsmanager_secret` | secret with outputs |

## Next steps after generation

1. **Fill in variables** — each `terragrunt.hcl` has an `inputs {}` block with
   `# TODO` comments. Replace them with your actual values.

2. **Initialise** — from the output directory:
   ```bash
   terragrunt run-all init
   ```

3. **Plan** — review what Terraform will create:
   ```bash
   terragrunt run-all plan
   ```

4. **Apply**:
   ```bash
   terragrunt run-all apply
   ```

## Adding a new cloud provider plugin

Implement the `model.Plugin` interface from [pkg/model/plugin.go](../pkg/model/plugin.go):

```go
type Plugin interface {
    Name() string
    Detect(files []model.SourceFile) ([]model.DetectedSDK, error)
    Map(sdk model.DetectedSDK) ([]model.ResourceSpec, error)
}
```

Register it in [cmd/groundwork/main.go](../cmd/groundwork/main.go):

```go
case "gcp":
    p, err := gcpplugin.New()
    // ...
    reg.Register(p)
```

The engine, registry and generator require no changes.
