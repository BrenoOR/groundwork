# Team Files API — exemplo

API REST em Python (FastAPI) que permite usuários acessarem seus arquivos pessoais e os arquivos compartilhados dos times aos quais pertencem, armazenados no S3.

## Estrutura do projeto

```
app/
├── domain/
│   ├── models.py          # User, Team, FileEntry
│   └── interfaces.py      # IUserRepository, IStorageRepository, IFileAccessService
├── repositories/
│   ├── user_repository.py     # InMemoryUserRepository
│   └── storage_repository.py  # S3StorageRepository (boto3)
├── services/
│   └── file_access_service.py # regras de acesso (pessoal + times)
├── api/
│   ├── dependencies.py        # injeção de dependências via FastAPI
│   └── routes/
│       ├── users.py           # GET /users/{user_id}
│       └── files.py           # GET /users/{user_id}/files
│                              # GET /users/{user_id}/files/{key}/download
└── main.py
```

## Layout de pastas no S3

```
<bucket>/
├── users/
│   ├── user-1/   ← pasta pessoal de Alice
│   ├── user-2/   ← pasta pessoal de Bob
│   └── user-3/   ← pasta pessoal de Carol
└── teams/
    ├── team-alpha/   ← compartilhada entre Alice e Bob
    └── team-beta/    ← compartilhada entre Bob e Carol
```

## Configuração

Copie o arquivo de variáveis de ambiente e ajuste conforme necessário:

```bash
cp .env.example .env
```

| Variável | Descrição | Padrão |
|---|---|---|
| `AWS_REGION` | Região AWS do bucket S3 | `us-east-1` |
| `S3_BUCKET` | Nome do bucket S3 com os arquivos | `my-company-files` |

## Rodar a API

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Documentação interativa disponível em `http://localhost:8000/docs`.

## Gerando a infraestrutura com groundwork

### Pré-requisitos

- [groundwork](https://github.com/groundwork-dev/groundwork) instalado
- [Terragrunt](https://terragrunt.gruntwork.io/) instalado
- Credenciais AWS configuradas (`aws configure` ou variáveis de ambiente)

### 1. Detectar recursos e gerar scaffold

A partir da raiz do repositório `groundwork`, execute:

```bash
groundwork \
  --input  ./examples/python_api \
  --output ./examples/python_api/infra \
  --state-bucket python-api-tf-state \
  --state-region us-east-1
```

Saída esperada:

```
groundwork: scanned 18 file(s) from "./examples/python_api"
groundwork: 1 resource spec(s) collected
groundwork: output written to "./examples/python_api/infra"
```

O groundwork detecta a chamada `boto3.client('s3')` em `app/repositories/storage_repository.py`
e gera a seguinte estrutura:

```
infra/
├── terragrunt.hcl              ← remote state no S3 com locking via DynamoDB
└── modules/
    └── s3/
        └── terragrunt.hcl      ← bucket S3 para os arquivos da aplicação
```

### 2. Pré-visualizar sem escrever arquivos (dry-run)

```bash
groundwork \
  --input  ./examples/python_api \
  --output ./examples/python_api/infra \
  --state-bucket python-api-tf-state \
  --dry-run
```

Saída esperada:

```
groundwork: scanned 18 file(s) from "./examples/python_api"
groundwork: 1 resource spec(s) collected
groundwork: dry-run — no files written
  ./examples/python_api/infra/terragrunt.hcl          (root remote_state)
  ./examples/python_api/infra/modules/s3/terragrunt.hcl  (aws_s3_bucket)
```

### 3. Conteúdo gerado

**`infra/terragrunt.hcl`** — remote state com S3 + locking via DynamoDB:

```hcl
# Generated by groundwork — do not edit manually.

remote_state {
  backend = "s3"

  config = {
    bucket         = "python-api-tf-state"
    key            = "${path_relative_to_include()}/terraform.tfstate"
    region         = "us-east-1"
    encrypt        = true
    dynamodb_table = "python-api-tf-state-lock"
  }

  generate = {
    path      = "backend.tf"
    if_exists = "overwrite_terragrunt"
  }
}
```

**`infra/modules/s3/terragrunt.hcl`** — módulo do bucket de arquivos:

```hcl
# Generated by groundwork — do not edit manually.
# Resource type: aws_s3_bucket

include "root" {
  path = find_in_parent_folders()
}

terraform {
  source = "tfr:///terraform-aws-modules/s3/aws///"
}

inputs = {
  # TODO: fill in required inputs for aws_s3_bucket
}
```

### 4. Aplicar a infraestrutura

```bash
cd examples/python_api/infra

# Aplica todos os módulos na ordem correta
terragrunt run-all apply
```

> **Atenção:** o bucket de state (`python-api-tf-state`) e a tabela DynamoDB
> (`python-api-tf-state-lock`) precisam existir antes do primeiro `apply`.
> Crie-os manualmente ou com `terragrunt apply` no módulo raiz primeiro.

### 5. Personalizar o módulo S3

Antes de aplicar, edite `infra/modules/s3/terragrunt.hcl` e preencha os inputs
necessários para o bucket de arquivos da aplicação. Exemplo:

```hcl
inputs = {
  bucket = "my-company-files"

  versioning = {
    enabled = true
  }

  server_side_encryption_configuration = {
    rule = {
      apply_server_side_encryption_by_default = {
        sse_algorithm = "AES256"
      }
    }
  }

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}
```
